A compiler for Latte programming language

This project was developed as an assignment in the 'Compiler construction' course at the University of Warsaw. The compiler was for code written in Latte (get its grammar from here) – a programming language which is a subset of Java.

I implemented the following features:

  • primary expressions and statements (loops, assignments, functions, if-else statements, etc.);
  • four types: int, string, boolean, void;
  • IO operations: printInt, printString, readInt, readString;
  • arrays;
  • full object-oriented programming support – structures, classes (attributes, methods, inheritence, virtual methods).

The front-end (a lexical and semantics analysis) and back-end (an assembly code generation) of the compiler were written in Haskell. Testing scripts were written in Python. A parser was generated by the bnfc tool.

The compiler outputs assembly code that can be run on x86 architectures. Programs written in Latte can be very easily translated to Java programs.

I contributed new functionality – the reference counting garbage collection. The source code is published and freely available on Github.

Amoose

BACHELOR'S THESIS, THE UNIVERSITY OF WARSAW

In a team of four students, we developed a mobile application for creating, viewing and sharing interactive 'amusements'. Amusements may consist of photos, sounds and simple interactions with the user. Gadgets are created in an intuitive graphical editor which is an Andorid app. Once an amusement is created, it can be stored on the server. The user is then able to send a link with the gadget to friends or share it on social networking services.

Graph drawer

The project is a demo that demonstrates HTML5 and AngularJS capabilities. It is a SPA which entirely runs in a web browser environment.

My purpose was to implement a few graph algorithms. In the current version, a user can enter graph directed edges as well as two distinguished nodes. Then, he can run one of the following algorithms that determine:

  • if a path exists from the starting point to the destination points;
  • if the graph in an Eulerian graph;
  • if the graph in a Hamiltonian graph.

The third algorithm can take a lot of time, so the algorithm computations are delegated to a web worker – a HTML5 feature that simulates a separate thread, not affecting the UI thread.

The graph is drawn on the canvas (another HTML5 feature). The location of the graph nodes are computed using an algorithm described in the paper …. The aim of this algorithm is to find locations of graph nodes on the plane so that it can be drawn a highly efficient way – reducing edges intersections, nodes overlapping, etc.

Another available feature is reading graph descriptions from the file and processing this file in the web browser (another HTML5 feature).

The source code is published on Github.
